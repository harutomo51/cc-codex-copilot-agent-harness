# Agent Harness Principles — 用語集

このプロジェクトのドキュメント・エージェント定義で参照される原則 ID の定義集。
原典は agent-essence の診断ルーブリック / ハーネス設計原則。

## C — Context / 帯域効率

- <a id="C-1"></a>**C-1 帯域ゼロサム** — コンテキストは有限資源。手順詳細を載せた分だけ原則の注意力が削られる。低頻度情報はポインタ化する。
- <a id="C-5"></a>**C-5 報酬ハッキング** — 評価の「ものさし」（スキーマ・検証スクリプト）を編集可能領域に置くと、エージェントはものさし自体を書き換えて pass を偽装しうる。
- <a id="C-6"></a>**C-6 排除宣言** — 「X ではなく Y」と肯定形で書くより、「X は使わない」と排除形で書くほうが誤用を防ぐ。
- <a id="C-7"></a>**C-7 校正盲** — 校正は自分の文章では機能しない。検証は常に独立した経路から行う。

## K — Knowledge / 知識設計

- <a id="K-2"></a>**K-2 エージェントからの可読性** — 原則 ID や略称を引用する場合、原典が同リポジトリ内から辿れるようにする。
- <a id="K-2.1"></a>**K-2.1 ポインタは百科事典より強い** — CLAUDE.md は地図として機能させ、詳細は別ドキュメントへリンクする。
- <a id="K-2.4"></a>**K-2.4 知識の出所明示** — 引用された事実・原則は出典への参照を持つ。

## T — Team / 委任設計

- <a id="T-2.3"></a>**T-2.3 統合判断を委任しない** — N エージェントの並行作業を整合させるには、統合判断を保持する単一ポイントが必要。整合判断をサブエージェントに渡すと矛盾が混入する。

## S — Security / 信頼境界

- <a id="S-1"></a>**S-1 評価と実装の権限分離** — 実装するエージェントが自分の評価基準を編集できてはならない。
- <a id="S-1.1"></a>**S-1.1 記憶の出所と露出先を追跡** — どの情報がどこから来てどこへ行くかを常にタグ付けする。
- <a id="S-1.3"></a>**S-1.3 最小権限で爆発半径を抑える** — 不可逆操作はデフォルト deny。
- <a id="S-1.4"></a>**S-1.4 防御は伝播停止の段階で評価する** — 入力検疫より、伝播停止のほうが確実。

## V — Verification / 検証ループ

- <a id="V-1.1"></a>**V-1.1 失敗を仕組みに昇格させる** — 「気をつけよう」を hook / deny / schema に変換する。
- <a id="V-2"></a>**V-2 フィードバックループを閉じる** — 検知のみで終わらせず、修正の入り口（fix 行・再書き込み手順）まで提供する。
